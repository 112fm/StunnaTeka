# Правила работы с приложением «Струнотека»

Этот файл — памятка по настройке и ежедневному использованию платформы. Так как проект использует внешние сервисы, важно понимать, как всё настроено.

## 1. Первоначальная настройка

Чтобы Струнотека "ожила" на новом устройстве или после очистки кэша браузера, необходимо зайти в **Настройки (⚙️)** и указать данные:

*   **GitHub Username:** Ваше имя пользователя на GitHub (например, `112fm`).
*   **GitHub Repository:** Название репозитория, куда складируются песни (например, `StunnaTeka`).
*   **GitHub Token (PAT):** Секретный токен для доступа. 
    *   *Важно:* Токен генерируется в браузере на сайте GitHub в разделе Developer Settings. Токен должен иметь права `repo` (чтение и запись репозиториев).
*   **Google Gemini API Key:** Ключ от Google AI Studio для работы функции "Распознать с помощью ИИ" по фото.

> ⚠️ Важно: Эти ключи сохраняются **только в вашем браузере** (в Local Storage). Они не передаются третьим лицам. Если вы зайдете с телефона, ключи потребуется ввести снова.

## 2. Логика работы с песнями

*   **Добавление:** При добавлении песни скрипт автоматически преобразует название песни в безопасное имя файла (например, `Moya_Oborona.json`) и отправляет на GitHub.
*   **Удаление и Изменение:** *В текущей архитектуре удаление в самом приложении искусственно усложнено или недоступно по соображениям безопасности случайных кликов.* Важно помнить, что мастер-копия базы лежит именно на GitHub. Чтобы удалить песню или изменить старые данные с нуля — зайдите на GitHub.com и удалите нужный `.json` файл из папки `songs/`.

## 3. Как пользоваться OCR (Распознаванием аккордов по фото)

Струнотека умеет вытаскивать гитарные разборы и аккорды из скриншотов благодаря нейросети.
1. На вкладке добавления песни найдите зеленую кнопку с иконкой фотоаппарата.
2. Вы можете:
   - Нажать и выбрать картинку(и) в открывшемся системном окне.
   - Сделать скриншот на компьютере, кликнуть мышкой внутрь окна Струнотеки и нажать `Ctrl + V` (Вставить).
3. Появится кнопка **"Распознать текст с помощью ИИ"**. 
4. Приложение само склеит ответы, если скриншотов несколько, и сохранит необходимые отступы для аккордов. Проверьте результат в текстовом поле и нажмите "Сохранить песню".

## 4. Разработка и Код

*   **Git:** Вы не должны делать изменения прямо в ветке `main` через веб-интерфейс Github во время активной разработки локально. Если вносятся правки в логику (`app.js`, `index.html`) на вашем ПК, они заливаются на GitHub обычными командами:
    ```bash
    git add .
    git commit -m "описание"
    git push
    ```
*   **Синхронизация:** Песни (JSON файлы) также пушатся в эту ветку. Поэтому при командной разработке не забывайте делать `git pull` перед внесением изменений в код, чтобы скачать на компьютер все новые `.json` файлы с песнями, которые были добавлены через приложение с телефона.
