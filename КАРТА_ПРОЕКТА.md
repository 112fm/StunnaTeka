# Карта проекта «Струнотека»

Данный документ описывает устройство проекта, основные файлы и то, как компоненты взаимодействуют друг с другом. 

## 📂 Структура файлов

Проект представляет собой **PWA (Progressive Web App)** — веб-приложение, которое работает в браузере, но может быть установлено на телефон или ПК как полноценная программа, работающая даже без интернета.

- **`index.html`**
  Отвечает за весь пользовательский интерфейс (UI). Содержит "скелет" приложения: всплывающие окна (модалки), три основные вкладки (Библиотека, Добавить песню, Настройки) и карточки песен.
  
- **`app.js`**
  "Мозг" приложения. Вся программная логика находится здесь.
  - **Роутинг/Вкладки:** Переключает видимость экранов (Библиотека, Форма добавления, Настройки).
  - **База данных (GitHub API):** Скачивает и загружает JSON-файлы песен напрямую в ваш репозиторий GitHub.
  - **Нейросеть (Gemini API):** Отправляет картинки в Google Gemini, получает распознанный текст с аккордами и вставляет в форму.
  - **Локальное состояние:** Хранит список уже загруженных песен в оперативной памяти смартфона/ПК для мгновенного поиска.
  
- **`styles.css`**
  Визуальное оформление. Цвета, отступы, шрифты, анимации. Здесь же настроена светлая и тёмная тема (`data-theme="dark"`).

- **`manifest.json`**
  Специальный конфигурационный файл для браузеров (Chrome, Safari). Благодаря ему браузер понимает, что это не просто сайт, а PWA-приложение, предлагает иконки и кнопочку "Установить на домашний экран".

- **`sw.js` (Service Worker)**
  Фоновый скрипт. Он перехватывает все сетевые запросы и кэширует файлы. Благодаря ему приложение мгновенно загружается (без белого экрана загрузки) и работает оффлайн (по крайней мере отрисовывает сам интерфейс).

## ⚙️ Как работают основные модули

1. **Хранение данных (GitHub)**
   Приложение не имеет классического "сервера" и базы данных (вроде MySQL). Вместо этого оно использует систему управления версиями GitHub. Когда вы нажимаете "Сохранить", скрипт генерирует `.json` файл и через GitHub REST API делает коммит (push) прямо в ваш репозиторий. По сути, GitHub выступает в роли бесплатного, вечного и надежного облачного хранилища.

2. **Распознавание (OCR)**
   Раньше использовалась сложная логика Tesseract, но теперь приложение связывается с мощной AI-моделью Google Gemini. Картинка переводится в формат Base64 и отправляется по сети на лету (вместе со строгим промптом, чтобы AI не удаляла пробелы). Приложение поддерживает "очередь" картинок — можно грузить несколько фото за раз.

3. **Сортировка и Поиск**
   Выполняется полностью на стороне клиента (в браузере). Это дает мгновенный, молниеносный отклик при вводе букв в строку поиска.
